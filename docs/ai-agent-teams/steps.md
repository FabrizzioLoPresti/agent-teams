# Paso a paso para inicializar equipos de agentes de IA

1. **Mantener actualizados archivos de documentación**: Actualizar regularmente los archivos de documentación del proyecto como `docs/architecture/` y `docs/decisions/` para reflejar los cambios en la arquitectura y las decisiones tomadas durante el desarrollo del proyecto y que la AI tenga un contexto claro de la evolución del proyecto.

2. **Refactorizar regularmente las reglas**: Refactorizar regularmente las rules definidas en `.claude/rules/` utilizadas por el agente para saber como debe comportarse, manteniendo el mismo formato y estructura del proyecto durante el desarrollo. Siempre deben ser reglas estructurales de componentes o de implementaciones, de forma generica similar a patrones de diseño, buenas prácticas, principios SOLID, etc. para mantener la coherencia del proyecto y asegurar que el agente pueda seguir las reglas de manera efectiva y desarrollar código de alta calidad.

3. **Creacion de skills con `/skill-creator`**: Generar las Skills necesarias para cada uno de los agentes dentro de `.claude/skills/` utilizando el comando `/skill-creator` para crear skills de manera estructurada y organizada, asegurando que cada skill tenga una función clara y específica dentro del proyecto, y que el agente pueda utilizarlas de manera efectiva para cumplir con sus tareas asignadas. Estas tendran funciones como authenticación, code review, creación de pruebas unitarias, componentes UI, endpoints de ORPC con scheamas de Zod y types, migraciones de bases de datos, etc. dependiendo de las necesidades del proyecto.

4. **Instalación de `Spec-Kit`**: Instalar el framework `Spec-Kit` para definir y ejecutar pruebas de especificación (mediante `Specify`) pudiendo mantener un desarrollo basado en pruebas y asegurando la calidad del código mediante SDD, PRD, TDD entre otras metodologías de desarrollo.

5. **Eliminacion y creacion de nuevos agentes basados en `Spec-Kit`**: Eliminar agentes obsoletos y crear nuevos agentes basados en `Spec-Kit` para ejecutar pruebas de especificación, asegurando que el equipo de desarrollo pueda validar su código de manera efectiva y mantener la calidad del proyecto. Ademas agregar a los mismos consideraciones para utilizar `skills (.claude/skills/)`, `rules (.claude/rules)` y `documentación (docs/architecture/ y docs/decisions/)` actualizada para mantener la coherencia del proyecto. **Luego de crear los agentes uno por uno actualizarlos** para que se apunten entre ellos ya que al crearlo uno a uno no se conocen entre ellos, por lo que es necesario actualizar cada agente para que se apunten entre ellos y puedan interactuar de manera efectiva, ademas de corregirlos para que se basen en la implementación de [Spec-Kit](https://github.com/github/spec-kit) desde Github. Crearlos o actualizarlos apuntando a sus respectivos modelos, con acceso unicamente a las herramientas que necesiten para cumplir su función, y con acceso a la documentación actualizada para que puedan entender el contexto del proyecto y desarrollar código de alta calidad.

6. **Actualizar el archivo `CLAUDE.md`**: Actualizar el archivo `CLAUDE.md` incluyendo referencias a nuevos `agentes`, `skills`, `rules` y `documentación (docs/architecture/ y docs/decisions/)` actualizada, asegurando que el equipo de desarrollo tenga acceso a la información más reciente sobre el proyecto y pueda entender cómo interactuar con los agentes de IA de manera efectiva. Definir siempre trabajar con "Agent Teams" de Claude Code para mantener la asincronia entre trabajo de agentes, siendo el Orquestador quien organice siempre que agente debe realizar cual tarea.

7. **Crear plugin**: Crear un plugin o repositorio conteniendo toda la estructura de agentes, skills, rules y documentación actualiza y de forma generica para ser utilizada dentro de cualquier proyecto futuro a desarrolloar, actualizada para que el equipo de desarrollo pueda acceder fácilmente a la información y recursos necesarios para interactuar con los agentes de IA de manera efectiva, asegurando que el proyecto se mantenga organizado y accesible para todos los miembros del equipo. Ademas de mantenerlo actualizado con los cambios realizados en los pasos anteriores para asegurar que el equipo tenga acceso a la información más reciente sobre el proyecto. Crear un simil libro de patrones de diseño, buenas prácticas, principios SOLID, etc. para mantener la coherencia del proyecto y asegurar que el agente pueda seguir las reglas de manera efectiva y desarrollar código de alta calidad.
